# Project History Notes

This note replaces several root-level progress documents that were useful during early migration work but were too noisy for the repository root.

## Topics previously tracked in standalone root files

- cleanup summaries after removing legacy code and build artifacts
- migration planning notes for the move toward the current Rust-first structure
- migration completion status and milestone summaries
- optimization reports covering duplicate logic cleanup, safety checks around unsafe code, and other targeted refactors

## Why this was consolidated

These notes are still part of the repository story, but they are historical records rather than primary onboarding documents. Consolidating them under `docs/archive/` keeps the root focused on the files new users actually need first.
